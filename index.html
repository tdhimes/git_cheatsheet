Git Cheatsheet - Sourced from Thinkful.com

***The following cheatsheet should serve as a reference for each of the steps in the basic git workflow***

How to commit your work for the first time in a new project:

Initialize a Repo
Type git init. The command line should say, "Initialized empty Git repository."


Check the Repo
Type git status. The command line should show you any untracked files.


Save Your Progress
Type git add. Add files one at a time.


Check What Changed
Type git status.


Commit Changes
Type git commit -m "{message}". Add files one at a time.


Success!
Type git status. You should see, "nothing to commit, working directory clean."


Pushing Your snapshot to GitHub:

Create GitHub Repo
Go to github.com and create a new repository.


Add New Remote Repo
Type git remote add origin git@github.com{custom}. Check GitHub for the specific SSH.


Send Changes to Repo
Type git push origin master. This command sends your changes to the repo.


Pull from Repo
Type git pull. Keeps local version up-to-date with the remote version.
